Operador Matriz Atual / Histórico


Operador matriz atual é utilizado para obter os valores da grade de um dado matricial, dinâmico ou estático.

II.1- Amostra

Retorna os valores da grade mais recente ou um histórico definido em relação à hora atual no caso de uma série de dados históricos de observação ou da camada mais recente de previsão. Para dados matriciais estático retorna os valores da grade própria grade.

SINTAXE GERAL:

grid.sample("<dynamic_data_grid>", [<band>], ["<historical_date>"])    

onde:

  • dynamic_data_grid : String com o nome da série de dados dinâmicos matriciais ou dados estáticos matriciais.
  • band : [Opcional] Banda da grade utilizada. Se não informado será considerado a primeira banda (0).
  • historical_date : [Opcional] Se declarado deve buscar pela data/hora especificado por uma sequência de string. A máscara a ser utilizada deve ser “%YYYY$2020%MM$04%DD$10%hh$00%mm$00%ss$00”, sendo possível passar somente parte dos parâmetros como ano/mês. Se não declarado será a grade de igual horário ou mais próxima do horário de execução.


Exemplo:        x = grid.sample("hidroestimador")        # matriz dinâmica (banda 0 por padrão)

               y = grid.sample("declividade")                # matriz estática (banda 0 por padrão)

               a1 = grid.sample("chuva", 1)                # matriz dinâmica (banda 1 – segunda camada do arquivo)

               a2 = grid.sample("chuva", 0, "%YYYY$1990%MM$04")

                                                            # matriz dinâmica no ano de 1990 e mês 04)


É possível também passar o valor de uma data de formar dinâmica. O exemplo a seguir:

ano = "2020"

mes = get_analysis_date().month

var1 = grid.sample("chuva", 0, "%YYYY$"+str(ano)+"%MM$"+str(mes).zfill(2))


ano = get_analysis_date().year

mes = get_analysis_date().month

date = "%YYYY$"+str(ano)+"%MM$"+str(mes).zfill(2)

var2 = grid.sample("chuva", 0, date)


OBS: Este código   str(mes).zfill(2)  significa que a string tem que ter 2 dígitos, ou seja, se for mês 1 deve ficar como 01, os que forem acima de 10 ficarão da mesma maneira sem alteração.
